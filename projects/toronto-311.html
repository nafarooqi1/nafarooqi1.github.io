Toronto 311 Service Request Prioritization

An XGBoost classification model on ~1 million Toronto 311 service requests, leveraging cyclical time encoding, PCA, and rolling backlog features to dynamically prioritize high-urgency requests.

Problem Statement

The City of Toronto receives hundreds of thousands of 311 service requests annually, ranging from pothole repairs to emergency safety concerns. Without intelligent prioritization, critical incidents risk being buried under routine requests — delaying response times and straining municipal resources.

This project aimed to build a data-driven prioritization system that automatically ranks incoming requests by urgency, enabling the city to allocate resources proactively and reduce response times for high-impact issues.

Methodology

  • Data Source: Toronto Open Data — 311 Service Requests (2024–2025), approximately 1 million records
  • Cyclical Time Encoding: Converted temporal features (hour, day, month) using sin/cos transforms to preserve cyclical nature for the model
  • Rolling Backlog Features: Engineered features tracking historical backlog per service type to reflect real-time operational pressure
  • Dimensionality Reduction: Applied PCA to reduce correlated features while retaining maximum variance
  • Model: XGBoost classifier with hyperparameter tuning via cross-validation
  • Hybrid Scoring: Final priority score = 70% ML prediction + 30% current backlog weight
