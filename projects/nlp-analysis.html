Project Overview

Named Entity Recognition (NER) is the task of identifying and classifying named entities in text into predefined categories such as person names, organizations, and locations. This project fine-tuned a pre-trained BERT (Bidirectional Encoder Representations from Transformers) model on the CoNLL-2003 benchmark - the industry-standard dataset for NER evaluation.

Rather than building a model from scratch, the approach leverages transfer learning: BERT was pre-trained on massive text corpora and then fine-tuned specifically for token-level classification on NER labels. This is a state-of-the-art deep learning approach used in real-world NLP applications like information extraction, knowledge graph construction, and document processing.

Why BERT for NER? Traditional NLP approaches (rule-based or feature-engineered models) struggle with context-dependent entity recognition. "Apple" could be a fruit or a company depending on context. BERT's bidirectional attention mechanism reads the full sentence in both directions simultaneously, allowing it to resolve such ambiguities accurately.

Entity Types Recognized

The model was trained to recognize 4 entity categories using BIO (Beginning, Inside, Outside) tagging - a standard NER annotation scheme with 9 labels total.

B-PER / I-PER
Person names

B-ORG / I-ORG

e.g. "Seneca Polytechnic", "Google"
B-LOC / I-LOC
Locations
e.g. "Toronto", "United Kingdom"
B-MISC / I-MISC
Miscellaneous entities
e.g. nationalities, events, products
# BIO label scheme - 9 labels total label_list = [ "O", # Outside any entity "B-PER", "I-PER", # Person (Beginning / Inside) "B-ORG", "I-ORG", # Organization "B-LOC", "I-LOC", # Location "B-MISC", "I-MISC" # Miscellaneous ] label2id = {lab: i for i, lab in enumerate(label_list)} id2label = {i: lab for lab, i in label2id.items()}

Dataset: CoNLL-2003

The CoNLL-2003 dataset is the industry-standard benchmark for NER evaluation, derived from Reuters news articles. It is pre-split into training, validation, and test sets in CoNLL format — one token per line with its NER label.

  • Format: CoNLL — each line contains a token and its BIO label, sentences separated by blank lines
  • Splits: Train / Validation / Test — already provided, no manual splitting required
  • Domain: News articles (Reuters wire stories)
  • Challenge: Subword tokenization — BERT's tokenizer splits words into subwords, requiring label alignment to preserve correct entity boundaries
# Load CoNLL format data def load_conll_data(filepath): sentences, labels = [], [] tokens, ner_tags = [], [] with open(filepath, "r", encoding="utf-8") as f: for line in f: line = line.strip() if not line: if tokens: sentences.append(tokens) labels.append(ner_tags) tokens, ner_tags = [], [] else: parts = line.split() tokens.append(parts[0]) ner_tags.append(parts[-1]) return sentences, labels train_sentences, train_labels = load_conll_data("/content/train.txt") valid_sentences, valid_labels = load_conll_data("/content/valid.txt") test_sentences, test_labels = load_conll_data("/content/test.txt")

Tokenization & Label Alignment

BERT uses a WordPiece tokenizer that splits words into subword units. For example, "Washington" might become ["Washington"] but "Farooqi" might become ["Far", "##ooqi"]. Each subword gets a position, but only the first subword of a word should carry the entity label - the rest are marked -100 to be ignored in the loss calculation.

MODEL_NAME = "bert-base-cased" tokenizer = BertTokenizerFast.from_pretrained(MODEL_NAME) MAX_LENGTH = 128 def align_labels_for_sentence(word_ids, token_labels): """ Aligns original token-level labels to BERT subword tokens. - Special tokens ([CLS], [SEP]) and padding -> -100 (ignored in loss) - First subword of a word -> original label - Subsequent subwords -> B-X converted to I-X """ label_ids = [] previous_word_idx = None for word_idx in word_ids: if word_idx is None: label_ids.append(-100) # special/padding token elif word_idx != previous_word_idx: label_ids.append(label2id[token_labels[word_idx]]) # first subword else: # convert B-X to I-X for continuation subwords orig = token_labels[word_idx] label_ids.append(label2id["I" + orig[1:]] if orig.startswith("B") else label2id[orig]) previous_word_idx = word_idx return label_ids

Model: Fine-tuning BERT

The bert-base-cased model (case-sensitive, important for NER) was loaded from HuggingFace and fine-tuned for token classification. A linear classification head was added on top of BERT's hidden states, mapping each token's representation to one of the 9 NER labels.

from transformers import BertForTokenClassification, Trainer, TrainingArguments from transformers import DataCollatorForTokenClassification # Load pre-trained BERT with NER classification head model = BertForTokenClassification.from_pretrained( "bert-base-cased", num_labels=len(label_list), # 9 labels id2label=id2label, label2id=label2id ) # Training configuration training_args = TrainingArguments( output_dir="./ner_bert", eval_strategy="epoch", save_strategy="epoch", learning_rate=5e-5, per_device_train_batch_size=16, per_device_eval_batch_size=16, num_train_epochs=3, weight_decay=0.01, load_best_model_at_end=True, metric_for_best_model="f1" ) trainer = Trainer( model=model, args=training_args, train_dataset=train_dataset, eval_dataset=valid_dataset, tokenizer=tokenizer, data_collator=DataCollatorForTokenClassification(tokenizer), compute_metrics=compute_metrics ) trainer.train()

Key Findings

  • F1 of 89.63% is strong performance on the CoNLL-2003 benchmark, which is the standard NER evaluation dataset used to compare models across the NLP research community
  • Recall (91.6%) higher than Precision (87.75%) means the model is slightly more aggressive at identifying entities - it misses fewer entities but occasionally tags non-entities. For most real-world NER applications, higher recall is preferred
  • Transfer learning is highly effective - BERT's pre-trained contextual representations allow the model to generalize to NER with just 3 epochs of fine-tuning on a relatively small labeled dataset
  • Case sensitivity matters - using bert-base-cased (rather than uncased) is important for NER because proper nouns like "Apple" vs "apple" carry different semantic meaning
  • Label alignment is the critical preprocessing step - incorrect subword-to-label alignment would corrupt the training signal and significantly reduce model performance
